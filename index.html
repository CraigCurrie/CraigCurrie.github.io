Chess Game With AI Player

During my third-year course, 'Foundations of Artificial Intelligence,' I was tasked with creating an AI player for the game of NIM using the MINIMAX algorithm. This experience sparked a strong interest in designing and implementing AI players. Motivated by this newfound interest, I decided to pursue a personal project to further explore and apply these concepts.

Python
▼

I landed on chess for a couple reasons. First, I had previously participated in chess tournaments a few years ago, which I enjoyed. Second, during discussions with the lecturer, it was mentioned that some of the concepts we were learning would be better suited to more complex games like chess.

Initially, I intended to implement the AI player using full Monte Carlo tree rollouts. However, this approach proved unfeasible due to the nature of chess. The issue was that Monte Carlo rollouts rely on simulating random moves until the game reaches an end state. Unfortunately, chess rarely reaches an end state(other than stalemate) when random moves are chosen, making this approach ineffective.

To overcome this setback and complete the project, I decided to implement a more advanced heuristic approach using the MINIMAX algorithm with a search depth of 3. One example of the heuristics I added was evaluating piece development based on how many squares are 'threatened.'

My prior experience with chess proved invaluable when evaluating the effectiveness of my AI player. Overall, I am quite pleased with how this project turned out. It gave me practical experience in adapting to unforeseen challenges, and it’s also something I could demonstrate to family members, who were able to use and interact with it.

Analysis of the VERIS Dataset

This project involved formulating and answering a research question by analyzing the VERIS dataset, which details data breaches. The specific question I explored was: "What impact does the time of year have on the attack surface or pattern of data attacks according to the VERIS dataset?".

Unlike my previous software development projects, this was primarily a research and data analysis project. I thoroughly enjoyed the change of pace and was proud to achieve full marks for my work.

Python Matlab Json Data analysis
▼

During my analysis, I discovered some interesting patterns and events. One notable finding was a large data breach involving an application called 'MoveIt.' I identified this anomaly while examining data plotted by month. This breach was so substantial that it heavily skewed the data.

To address this issue, I re-ran the query, omitting all breaches that referenced 'MoveIt' in the summary and occurred in June 2023—the month of the breach. This adjustment produced more usable data, which I then plotted using MATLAB. One key observation was a spike in 'Error' incidents (e.g., carelessness or accidental actions, such as handing out personal information to the wrong recipient) during the summer months.

After conducting further research, I was able to link this trend to a phenomenon in the medical field known as the 'July Effect.' This effect occurs when an influx of new medical residents and related factors lead to an increase in errors in hospitals and medical practices.

I particularly enjoyed the change of pace this research project offered compared to my usual software development-focused projects.

This project was particularly challenging for two main reasons. First, it heavily focused on concurrency, which was a concept I was relatively new to. Second, there were difficulties with group members not fully participating.

The objective of the project was to develop concurrency-safe messaging software—similar in functionality to the app 'Discord'—using Java.

Java Concurrency

Walking Pacer Web App

This project involved developing a web app with a pace calculator, which performed a simple calculation based on data manually inputted by the user. The second part of the project focused on implementing a location tracker, allowing users to track and calculate the pace of their walks or runs in real time.

HTML CSS JavaScript
▼

This project was relatively straightforward to implement, though it became more complex when developing the location tracker. This was my first experience accurately calculating distance moved and handling device permissions, which presented some initial challenges.

One significant hurdle I encountered was testing. While I could easily test the app by simulating sensor data, which produced correct results. However physical testing revealed issues, it was hard to pinpoint the cause of this but I eventually was able to narrow it down to a spotty connection.

To address this, I began by hypothesizing why the app was returning incorrect distances. I eventually concluded that poor connections led to less precise location data, causing the reported position to jump around within a small area even when the device hadn’t actually moved. This was fitting since the incorrect results were always greater than expected.

To resolve this, I modified my code to update the distance and starting location only when the device had moved beyond a certain threshold from the initial position. This adjustment largely resolved the issue and improved the app’s reliability during real-world testing.

Ultimately, I achieved a mark of 91% for this project, which I’m very proud of.
